FT900 IoT Cloud Platform

Bridgetek releases IoT cloud platform to easily bridge FT900 series of microcontrollers to the cloud.

This IoT platform is not your usual IoT platform. It focuses on remote device access and control. For sensor data graphing and data analysis, use the IoT platforms of AWS, GCP and Azure.

Test procedure:
  1. Create an account then login
  2. Register a device (can register multiple devices)
  3. Copy the device information (DEVICE_ID, TLS_CERT and TLS_PKEY)
  4. Run the device (using the DEVICE_ID, TLS_CERT, TLS_PKEY and Bridgetek TLS_ROOT_CA)
  5. Access and control the device (set/get gpio/rtc, get ip/subnet/gateway/mac addresses, write uart)
Device credentials/certificates:
  • Host: ec2-13-229-115-250.ap-southeast-1.compute.amazonaws.com
  • Port: 8883 (MQTT over TLS) or 5671 (AMQP over TLS)
  • Client id: DEVICE_ID
  • Username: NULL
  • Password: NULL
  • Root CA certificate: TLS_ROOT_CA
  • Certificate: TLS_CERT
  • Private key: TLS_PKEY
  • Subscribe topic: DEVICE_ID/# (MQTT) or mqtt-subscription-DEVICE_IDqos1 (AMQP)
  • Publish topic: server/DEVICE_ID/API_NAME (MQTT) or server.DEVICE_ID.api (AMQP)

  • FT900 device
  • MQTT/AMQP device simulators (both Python and NodeJS)

FT900 IoT Device Framework

Bridgetek officially release FT900 IoT device framework to easily and securely bridge FT900 series of microcontrollers to the cloud.

FT900 can securely connect to FT900 IoT Cloud Platform and can also connect to AWS/GCP/Azure IoT cloud platforms by simply creating an IoT project via our customized Eclipse IDE with FT900 toolchain.

IoT platforms of popular cloud providers:
  • Amazon Web Services - IoT Core
  • Google Cloud Platform - IoT Core
  • Microsoft Azure - IoT Hub
FT900 has also been tested to work with popular IoT platform providers listed below. Simple configuration file changes are needed.

Dedicated IoT platform providers:
  • Adafruit.io
  • Ubidots
  • ThingSpeak
