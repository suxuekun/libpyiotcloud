User Guides

  1. REGISTER A DEVICE. This will produce a deviceid, certificate, private key and rootca. Use these information on FT900 MCU device.
  2. RUN THE DEVICE. Use the information retrieved from step 1.
  3. CONTROL THE DEVICE. If the device is running, you'll be able to access and control MCU details such as Ethernet, GPIO, RTC and UART, including information from connected sensors such as temperature, humidity, volume, brightness and color.

Device Connectivity

  • HOST: richmondu.com
  • PORT: 8883 (MQTT)
  • USERNAME: NULL
  • PASSWORD: NULL
  • CLIENTID: deviceid returned during registration
  • TLS ROOTCA: returned during registration
  • TLS CERT: returned during registration
  • TLS PKEY: returned during registration
  • SUBSCRIBE: deviceid/#
  • PUBLISH: server/deviceid/api

If using AMQP protocol instead of MQTT, use the following:

  • PORT: 5671 (AMQP)
  • SUBSCRIBE: mqtt-subscription-deviceidqos1
  • PUBLISH: server.deviceid.api

Device Processing

  1. Subscribe to MQTT topic "deviceid/#"
  2. Receive MQTT payload with topic "deviceid/api"
  3. Parse the api
  4. Process the api with the given payload
  5. Publish answer to topic "server/deviceid/api"
